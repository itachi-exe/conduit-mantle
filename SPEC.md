# CONDUIT — Fully Functional Onchain Research Agent for Mantle

Build CONDUIT — a FULLY FUNCTIONAL, end-to-end onchain research agent for Mantle.
This is NOT a mockup, demo, or static UI. Every number on screen must come from real
live Mantle chain data, and every agent brief must be generated by a real LLM call at
runtime. If it's fake, it's wrong. Build the whole working system.

*(Supersedes the earlier mock-data version of this spec, preserved in git history.)*

## What "working" means (non-negotiable)

- Real data: pull live from Mantle mainnet (RPC + protocol contracts + a data source
  like DefiLlama/Mantle explorer APIs). No hardcoded numbers.
- Real agent: briefs are generated at runtime by an actual model call that receives the
  live ranked signals as input and writes the interpretation. No pre-written strings.
- Real pipeline: data → signal detection → agent → surface, all wired and running.
- mockData is allowed ONLY as a fallback when an API is down, and must be clearly
  labeled as fallback in code. Default path is always live.

## Architecture (4 layers, all real)

1. **Data layer** (backend, Next.js API routes) — connect to Mantle mainnet via
   viem + public RPC. Pull TVL and protocol-level data from DefiLlama. Normalize into
   one schema. Cache with a short TTL so the demo is fast but data is fresh.
2. **Signal layer** — a real function that diffs current vs prior snapshot, computes
   deltas, and ranks changes by significance (magnitude × unusualness). Outputs a
   ranked signal list.
3. **Agent layer** — a real backend endpoint that takes ranked signals + asset context
   and calls an LLM to write the brief (what's happening / why it matters / what to
   watch). Wire in Mantle's AI Agent Skills as the interpretation layer if a real
   SDK/docs exist; otherwise build the interface so it's a one-line swap once they do,
   and disclose the gap honestly rather than faking the integration. API keys via
   .env, never hardcoded.
4. **Surface layer** (React + Tailwind) — consumes the backend endpoints. Renders
   live snapshot, signal, asset views, and generated briefs.

Match the look, feel, and branding of x.ai — near-black background, tight-tracked
large hero headline, thin high-contrast sans type, mono labels/status chips, glass
card panels, inline product-preview cards, big-number stats row, multi-column footer.
Build original assets — no copied images, logos, or copyright text.

## Page structure (mirror x.ai section order)

NavBar (Conduit wordmark, "Mantle · live" chip, Open Dashboard btn) → Hero ("Read the
chain. Not just the charts." + inline live Agent Brief preview) → Ecosystem Snapshot
(3 live metric cards + full-width live Agent Brief panel) → The Signal (top ranked
shift + live agent interpretation) → Asset View (protocol list, live trend chart,
composability classification) → Per-Asset Agent Brief (3 blocks + live-generated
badge) → Stats Row → Footer.

## Stack

Next.js (frontend + API routes in one repo) + Tailwind + Recharts + viem.
Anthropic API on the backend. `.env` for all keys/RPC URLs.
