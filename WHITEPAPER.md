# CONDUIT

### The Onchain Research Agent for Mantle

**Whitepaper, v1.0**

---

## Abstract

Conduit is a Mantle-native research system that converts raw onchain data into
interpreted signal. It combines a live data pipeline (Mantle RPC and
DefiLlama), a significance-ranking layer, and a dual-provider LLM agent to
answer, in under a minute, a question that today takes a researcher hours of
manual cross-referencing to answer: *what is actually happening on Mantle
right now, and why does it matter.* Conduit is delivered as a web dashboard,
a conversational research agent with live tool-use, a terminal CLI, and a
portable skill package any LLM can load. Every figure it displays is fetched
at request time; every interpretation is generated by a live model call.
Nothing on the default path is pre-written or fabricated.

---

## 1. The Problem

Onchain data is public. Onchain understanding is not.

Reading the actual state of a chain like Mantle today means manually
stitching together explorers, protocol dashboards, and ad hoc queries, then
forming a judgment by hand. The friction is not access, it is synthesis. A
researcher can see that a protocol's TVL moved, but not whether that move is
routine noise or the most significant thing that happened on the chain that
week. This gap is where time is lost, and where most participants who are not
full-time analysts simply stop looking.

Conduit closes that gap by doing what a human researcher does, at machine
speed: it watches the ecosystem, detects what changed, ranks it by
significance, and explains it in plain language, with the option to ask it
anything further and have it go look.

---

## 2. Design Principles

Conduit is built against three non-negotiable constraints:

1. **Live data only.** Every number shown is fetched from Mantle's public
   RPC or DefiLlama's public API at request time. There is no hardcoded or
   pre-computed figure on the default path.
2. **Live inference only.** Every brief and every chat reply is generated by
   a model call that receives the current data as input. No response is a
   stored string.
3. **Disclosed provenance, disclosed degradation.** Every claim traces to a
   source link. If a live source is unreachable and the system has to fall
   back to a cached or bundled snapshot, that is shown as a visible banner,
   not silently absorbed.

---

## 3. System Architecture

Conduit is organized into four layers.

### 3.1 Data Layer

A Next.js API layer reads Mantle mainnet (chain ID 5000) directly over its
public RPC endpoint via `viem`, block height, block timestamp, and gas
price, refreshed on every snapshot. In parallel, it pulls chain-wide and
per-protocol historical TVL from DefiLlama, reading each tracked protocol's
`chainTvls.Mantle` slice specifically rather than its global multichain
figure, so a protocol's Mantle-specific health is never diluted by unrelated
activity on other chains it also happens to be deployed on. Results are
cached with a short TTL to keep the dashboard responsive without hammering
either upstream source, with a stale-then-fallback degradation path if a live
fetch fails.

### 3.2 Signal Layer

A pure function takes the current and prior snapshot and produces a ranked
list of what changed and how much it matters, detailed in Section 4.

### 3.3 Agent Layer

Two LLM providers are used deliberately, for different jobs:

- **DeepSeek** generates the high-frequency, low-stakes dashboard briefs
  (ecosystem summary, top signal, per-asset writeups), cheap enough to
  regenerate often.
- **Claude** powers the conversational research agent, where tool-calling
  and live web search justify the higher cost per turn.

Each provider is the other's fallback: if Claude is unreachable mid-chat, the
turn retries on DeepSeek (without tool-use, since DeepSeek has no hosted
search); if DeepSeek is unreachable for a dashboard brief, it retries on
Claude. The system only reports failure to the user if both are unreachable.

### 3.4 Surface Layer

Four distribution surfaces consume the same underlying pipeline: the web
dashboard, the conversational agent, a terminal CLI, and a portable LLM skill
package. See Section 7.

## 4. Methodology

### 4.1 Signal Ranking

Raw percentage change is a poor significance measure on its own, a large,
stable protocol drifting 1% is noise, while a small, usually-quiet protocol
moving three times its typical weekly swing is a real signal. Conduit scores
each tracked protocol as:

```
significance = |7-day % change| × unusualness
unusualness  = |7-day % change| ÷ (that protocol's own trailing daily
               volatility × √7), capped at 8×
```

Volatility is computed per protocol from its own trailing daily return
series, so a naturally volatile DEX and a naturally stable lending market are
each judged against their own baseline, not a shared one. The top-ranked
result becomes "the signal" surfaced on the dashboard.

### 4.2 Composability Classification

Tracked protocols are split into capital that is actively deployed to trade
or earn (Dexs, Lending) versus capital that is held or passively exposed
(RWA), using DefiLlama's own category taxonomy rather than an invented
metric. This produces an active-versus-idle ratio for the ecosystem as a
whole, a disclosed, reproducible methodology, not an editorialized score.

## 5. The Research Agent

### 5.1 Dashboard Briefs

Ecosystem-, signal-, and asset-level briefs are short, plain-English
writeups generated at load/request time from the current ranked data,
rendered with a typewriter reveal rather than appearing all at once.

### 5.2 Conversational Agent

The chat surface runs a genuine agentic tool-use loop, not a single
context-stuffed completion. On each turn, Claude may call:

- **`web_search`**, Anthropic's hosted search tool, for anything outside
  Conduit's own data.
- **`get_ecosystem_snapshot`**, re-pulls live snapshot or RPC heartbeat data
  on demand, for the freshest possible read.
- **`search_mantle_protocols`**, searches every protocol DefiLlama tracks on
  Mantle, not only the curated set, so the agent can answer about a protocol
  it was not pre-loaded with.
- **`get_protocol_detail`**, fetches live TVL detail for any single Mantle
  protocol by slug, with its DefiLlama source link.

Each tool call is streamed to the client as a visible status line before its
result returns, the research process is observable, not a black box. Every
URL the turn touches (search results, DefiLlama links) is collected and
rendered as a clickable source list at the end of the reply, and each reply
is tagged with a topic label determined deterministically from the real
protocol registry, not asked of the model, so the label can never drift from
what the answer is actually grounded in.

### 5.3 Reliability

If Claude is unavailable mid-turn and nothing has streamed yet, the turn
retries on DeepSeek with a quiet, plainly-worded note that live search and
extra lookups were not available for that answer. If neither provider is
reachable, the agent says so directly rather than guessing.

## 6. Access Control

### 6.1 Wallet Gate

The conversational agent requires a connected wallet before it will respond.
Any injected, EIP-1193-compatible wallet (MetaMask, Rabby, OKX, and similar)
is supported directly, there is no WalletConnect step and no signature
requested. Connecting supplies only an address, used as an identifier for
rate-limiting; it is not a proof-of-ownership or authentication mechanism,
and should not be treated as one for anything beyond metering a free
feature.

### 6.2 Rate Limiting

Each wallet address is limited to ten chat messages per rolling UTC day,
enforced server-side and reflected live in the chat interface as the count
decrements. Requests without a valid connected address are rejected before
reaching the model.

## 7. Distribution Surfaces

Conduit is not confined to a browser tab.

- **Web dashboard**, the primary surface: live ecosystem snapshot, ranked
  signal, per-asset drilldown, and the conversational agent.
- **Terminal CLI** (`npx conduit-mantle`), the identical agent and tool-use
  logic, running as a terminal REPL against a fresh live snapshot on launch.
- **LLM skill package**, a dependency-free Node script (no `npm install`, no
  API key; raw `eth_*` JSON-RPC over `fetch` in place of a client library)
  that ports the read-only half of the pipeline, chain heartbeat, TVL
  snapshot, signal ranking, composability, protocol search and detail, into
  a form any LLM or agent runtime that can execute a script can call
  directly, mid-conversation, without going through Conduit's own servers.

All three surfaces share the same underlying data and agent logic; none is a
reduced or divergent implementation of another.

---

## 8. Current Scope

Conduit today tracks six near-native Mantle protocols in depth (Merchant
Moe, Agni Finance, Fluxion Network, INIT Capital, Lendle, and Mantle Index
Four Fund) with full historical TVL and signal ranking, plus on-demand search
and detail lookup across every other protocol DefiLlama tracks on Mantle.
The composability methodology classifies capital as active or passive at the
category level.

Work not yet built, and not claimed as built: per-wallet behavioral
analytics (accumulation/rotation/exit patterns), a narrative-versus-reality
sentiment overlay, treasury-activity correlation tracking, and drilldowns
into specific named tokenized real-world assets beyond what DefiLlama's own
categorization already surfaces. These are natural extensions of the same
architecture, the data layer, signal layer, and agent layer already
generalize to them, but they are future work, not present functionality.

---

## 9. Conclusion

Conduit treats Mantle's onchain activity as something to be read, not just
displayed. The distinction is architectural, not cosmetic: a live data
pipeline feeds a disclosed significance methodology, which feeds a
tool-using agent that can go verify anything it is unsure of, across four
surfaces that all share the same underlying truth. The result is a system
that a researcher can query once and trust, because every number in front of
them was fetched, not remembered, and every sentence around it was written
just now, not stored.
